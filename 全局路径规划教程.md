# 全局路径规划教程

本文是本工程推荐的全局路径规划流程：Gazebo 仿真 + 静态地图 + GPS/EKF 自动定位 + Nav2 全局路径规划。

当前默认地图：

```bash
~/yildiz_ws/src/YILDIZ-USV/maps/roboboat_manual_map_2_nav.yaml
```

## 1. 每个终端先执行

```bash
cd ~/yildiz_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
```

如果通过 SSH 输入命令，但显示器接在 Jetson/Nano 本机，启动 Gazebo 或 RViz 的终端先执行：

```bash
export DISPLAY=:0
```

如果仍报 `could not connect to display`，在接显示器的本机桌面终端执行一次：

```bash
xhost +SI:localuser:jetson
```

如果 GUI 能打开但窗口黑屏、卡住或出现 Qt/X11 共享内存相关异常，再尝试：

```bash
export QT_X11_NO_MITSHM=1
```

如果使用 VNC、远程桌面或 `ssh -X/-Y`，先看 `echo $DISPLAY`，不要强制改成 `:0`。

## 2. 启动 Gazebo

```bash
export DISPLAY=:0
ros2 launch workspace_gz simulation.launch.py
```

无桌面环境时：

```bash
ros2 launch workspace_gz simulation.launch.py headless:=true
```

## 3. 启动定位融合

```bash
ros2 launch workspace_ros localization.launch.py publish_map_to_odom:=false
```

这里不能开启固定 `map->odom`，本流程的 `map->odom` 由 GPS/EKF 自动生成。

## 4. 启动静态地图和规划器

```bash
ros2 launch workspace_nav global_planner.launch.py \
  use_gps_map_tf:=true
```

该 launch 会启动：

- `map_server`
- `planner_server`
- `gps_map_tf`
- `rviz_goal_planner`
- `/planned_global_path` 路径发布

## 5. 启动 RViz

```bash
export DISPLAY=:0
ros2 launch workspace_nav nav_rviz.launch.py
```

RViz 中应看到：

- 静态地图 `/map`
- 实时雷达 `/roboboat/sensors/lidar/scan`
- 船模型和 TF
- 规划路径 `/planned_global_path`

## 6. 发送规划目标

方式一：RViz 点目标

在 RViz 工具栏选择 `2D Goal Pose`，在地图上点终点。系统会自动调用规划器，并显示绿色全局路径。

方式二：命令行目标

```bash
ros2 run workspace_nav global_path_client \
  --goal-x 50.0 \
  --goal-y 0.0 \
  --goal-yaw 0.0
```

## 7. 启动自主路径跟踪

全局规划本身仍然只负责发布 `/planned_global_path`，不会直接控制船。需要自主航行时，另开终端启动路径跟踪节点：

```bash
ros2 launch workspace_nav path_follower.launch.py
```

该 launch 会启动：

- `path_follower`：订阅 `/planned_global_path`，根据 `map -> base_link` TF 沿路径跟踪
- `converter`：订阅 `/cmd_vel_nav`，转换成左右推进器推力

如果你已经单独启动了 `converter`，可以这样避免重复启动：

```bash
ros2 launch workspace_nav path_follower.launch.py start_converter:=false
```

停止 `path_follower` 后会发布一次零速度。没有启动该节点时，系统只显示和发布规划路径，不会自动航行。

不要同时运行 `manual_control` 和 `path_follower`，避免两个节点同时控制推进器。

常用调参：

```bash
ros2 launch workspace_nav path_follower.launch.py \
  lookahead_distance:=2.5 \
  max_linear_speed:=1.0 \
  max_angular_speed:=0.45
```

默认参数文件：

```bash
~/yildiz_ws/src/YILDIZ-USV/workspace_nav/config/path_follower.yaml
```

## 8. 手动控制船

如果只启动了全局规划、没有启动 `path_follower`，仍然可以使用手动控制。

另开终端：

```bash
ros2 run workspace_ros manual_control
```

按键：

- `W` 前进
- `A` 左转
- `D` 右转
- `S` 停止
- `Q` 退出

注意：如果启动了自动路径跟踪控制，就不要同时运行 `manual_control`，避免两个节点同时控制推进器。

## 9. 不建议在 Gazebo 里直接拖船

直接拖船等于瞬间传送模型，EKF 会短时间不连续，RViz 的 `Filtered Odometry` 可能出现一圈历史轨迹。

如果必须拖船：

```bash
# 1. 先关闭 localization、global_planner、rviz
# 2. 在 Gazebo 里移动船
# 3. 再重新启动定位、规划器和 RViz
```

不要拖完后继续沿用旧的 EKF 状态。

## 10. 常见检查

检查船在 map 中的位置：

```bash
ros2 run tf2_ros tf2_echo map base_link
```

检查规划 action 是否存在：

```bash
ros2 action list | grep compute_path_to_pose
```

检查路径话题：

```bash
ros2 topic echo /planned_global_path --once
```

检查自动跟踪速度输出：

```bash
ros2 topic echo /cmd_vel_nav
```

如果 RViz 没有路径：

- 确认 `use_gps_map_tf:=true`
- 确认目标点在地图自由区域内
- 确认 `/planned_global_path` 显示已勾选
