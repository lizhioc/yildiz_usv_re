# RoboBoat 手动航行 SLAM 建图教程

本文说明如何在不使用相机、不启动 Nav2 的情况下，通过手动控制 RoboBoat 航行，并使用 LiDAR + SLAM 实时建图，最后保存地图文件。

适用路径：

```bash
/home/jetson/yildiz_ws/src/YILDIZ-USV
```

工作空间路径：

```bash
/home/jetson/yildiz_ws
```

## 1. 当前建图链路说明

手动建图模式下，核心链路如下：

```text
Gazebo 仿真
  -> GPS / IMU / LiDAR
  -> garden_bridge
  -> ROS 2 传感器话题

GPS + IMU
  -> robot_localization
  -> /odometry/filtered
  -> TF: odom -> base_link

LiDAR + odom -> base_link
  -> slam_toolbox
  -> /map
  -> RViz 实时显示
  -> map_saver_cli 保存地图
```

建图时不使用相机，也不需要启动以下节点：

```text
target_buoy
kamikaze
Nav2
```

## 2. 建图前先清理旧进程

如果之前启动过仿真、SLAM、RViz、Nav2，建议先清理，避免旧 `/map` 或旧 TF 残留导致 RViz 画面跳动。

执行：

```bash
pkill -f "workspace_slam" || true
pkill -f "slam_toolbox" || true
pkill -f "rviz2" || true
pkill -f "workspace_nav" || true
pkill -f "workspace_ros" || true
pkill -f "workspace_gz" || true
pkill -f "gz sim" || true
pkill -f "manual_control" || true
pkill -f "converter" || true
pkill -f "robot_state_publisher" || true
pkill -f "joint_state_publisher" || true
pkill -f "static_transform_publisher" || true
pkill -f "odom_tf_publisher" || true
```

刷新 ROS 2 daemon：

```bash
cd ~/yildiz_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 daemon stop
ros2 daemon start
```

确认是否清理干净：

```bash
pgrep -af "rviz2|slam_toolbox|workspace_slam|workspace_gz|workspace_ros|workspace_nav|gz sim|map_server|amcl|robot_state_publisher|joint_state_publisher|static_transform_publisher|odom_tf_publisher"
```

如果没有输出，说明进程层面基本干净。

再检查 ROS 图：

```bash
ros2 node list
ros2 topic info -v /map
ros2 topic info -v /tf
ros2 topic info -v /tf_static
```

干净状态下，这些通常没有旧的 publisher。

## 4. 终端 1：启动 Gazebo 仿真

```bash
cd ~/yildiz_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
export DISPLAY=:0
ros2 launch workspace_gz simulation.launch.py
```

启动后，Jetson 桌面上应出现 Gazebo 窗口。

检查 RoboBoat 是否存在：

```bash
gz model --list
```

应能看到：

```text
roboboat
```

检查传感器是否有 ROS 话题：

```bash
ros2 topic list | grep roboboat
```

常见话题包括：

```text
/roboboat/sensors/gps/navsat
/roboboat/sensors/imu/imu
/roboboat/sensors/lidar/scan
```

## 5. 终端 2：启动定位，使用建图模式

建图时必须关闭 localization 里的静态 `map -> odom`，让 `slam_toolbox` 来发布 `map -> odom`。

执行：

```bash
cd ~/yildiz_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch workspace_ros localization.launch.py publish_map_to_odom:=false
```

注意，建图模式不要使用：

```bash
ros2 launch workspace_ros localization.launch.py
```

因为默认模式会发布静态 `map -> odom`，可能和 SLAM 的 `map -> odom` 冲突。

检查 EKF 输出：

```bash
ros2 topic echo --once /odometry/filtered
```

检查 TF：

```bash
ros2 run tf2_ros tf2_echo odom base_link
```

如果能持续输出位置和姿态，说明 `odom -> base_link` 正常。

## 6. 终端 3：启动 SLAM 和 RViz

执行：

```bash
cd ~/yildiz_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch workspace_slam slam.launch.py
```

该启动文件默认会打开 RViz，并把显示设置为 Jetson 桌面：

```text
DISPLAY=:0
```

如果你想明确指定显示器：

```bash
ros2 launch workspace_slam slam.launch.py display:=:0
```

如果只想后台建图，不打开 RViz：

```bash
ros2 launch workspace_slam slam.launch.py rviz:=false
```

RViz 里默认显示：

```text
/map
/roboboat/sensors/lidar/scan
TF
RobotModel
/odometry/filtered
```

## 7. 检查 SLAM 是否正常

检查 `/map` 发布者：

```bash
ros2 topic info -v /map
```

建图模式下，应该看到 `/map` 只有一个 publisher，并且节点名是：

```text
slam_toolbox
```

检查是否有错误的 `map_to_odom_tf`：

```bash
ros2 node list | grep map_to_odom
```

建图模式下最好没有输出。

检查 `map -> base_link`：

```bash
ros2 run tf2_ros tf2_echo map base_link
```

如果能持续输出，说明：

```text
map -> odom -> base_link
```

这条 TF 链是通的。

检查 LiDAR：

```bash
ros2 topic hz /roboboat/sensors/lidar/scan
```

检查地图内容：

```bash
ros2 topic echo --once /map
```

如果看到类似：

```text
width: 1102
height: 639
resolution: 0.05
```

说明 SLAM 已经生成了占据栅格地图。

其中：

```text
width / height 是地图栅格尺寸
resolution 是每个栅格代表多少米
```

例如：

```text
width: 1102
height: 639
resolution: 0.05
```

表示地图覆盖范围约为：

```text
1102 * 0.05 = 55.1 m
639  * 0.05 = 31.95 m
```

## 8. 终端 4：启动手动控制

手动控制节点需要在真实交互终端里运行。

执行：

```bash
cd ~/yildiz_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 run workspace_ros manual_control
```

按键说明：

```text
W：前进
A：左转
D：右转
S：停止
Q：退出手动控制
```

建议刚开始不要跑太快。可以先短按 `W`，观察 Gazebo 和 RViz 中船是否移动，再用 `S` 停止。

推荐建图方式：

```text
1. 先沿浮标区域慢速前进
2. 适当左右转向，让 LiDAR 扫到更多障碍物
3. 尽量形成闭环路线，例如绕一圈回到附近
4. 避免长时间只在开阔水面直线航行
```

水面太空旷时，LiDAR 扫不到足够特征，SLAM 地图质量会下降。

## 9. 保存地图

当 RViz 中地图已经比较完整后，另开一个终端执行：

```bash
cd ~/yildiz_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
mkdir -p ~/yildiz_ws/maps
ros2 run nav2_map_server map_saver_cli -f ~/yildiz_ws/maps/roboboat_manual_map
```

保存成功后会生成：

```text
~/yildiz_ws/maps/roboboat_manual_map.yaml
~/yildiz_ws/maps/roboboat_manual_map.pgm
```

查看文件：

```bash
ls -lh ~/yildiz_ws/maps/roboboat_manual_map.*
```

其中：

```text
.yaml：地图元数据，包括分辨率、原点、阈值、图像文件路径
.pgm：占据栅格地图图像
```

## 11. 常见问题

### 11.1 RViz 没打开

检查是否有显示器：

```bash
echo $DISPLAY
```

如果为空，手动设置：

```bash
export DISPLAY=:0
```

也可以启动时指定：

```bash
ros2 launch workspace_slam slam.launch.py display:=:0
```

### 11.2 RViz 画面在旧地图和新地图之间跳

通常是旧进程没有杀干净，或者有两个 `/map` / `map -> odom` 来源。

检查 `/map`：

```bash
ros2 topic info -v /map
```

建图模式下应该只有：

```text
Node name: slam_toolbox
Publisher count: 1
```

检查是否有 `map_to_odom_tf`：

```bash
ros2 node list | grep map_to_odom
```

建图模式下最好没有输出。

如果有残留，回到第 2 节重新清理。

### 11.3 RobotModel 显示 Error

通常是 `/robot_description` 或 `robot_state_publisher` 没起来。

检查：

```bash
ros2 topic echo --once /robot_description
ros2 node list | grep robot_state_publisher
```

如果没有，检查仿真是否正常启动。

### 11.4 TF 显示 Warn

先检查：

```bash
ros2 run tf2_ros tf2_echo map base_link
```

如果失败，说明 `map -> odom -> base_link` 不通。

常见原因：

```text
localization 没启动
slam_toolbox 没启动
localization 没加 publish_map_to_odom:=false
旧 TF 发布节点残留
```

### 11.5 manual_control 启动后不能按键

`manual_control` 必须在真实交互终端运行，不能放到后台，也不要用非交互脚本启动。

正确方式：

```bash
ros2 run workspace_ros manual_control
```

然后直接在该终端里按：

```text
W / A / S / D / Q
```

### 11.6 地图一直是空的

检查 LiDAR 是否有数据：

```bash
ros2 topic hz /roboboat/sensors/lidar/scan
```

检查 scan 中是否有有效距离，不应该全是 `.inf`。

如果在非常空旷的水面，LiDAR 没扫到浮标、岸边、障碍物，SLAM 地图会很难增长。建议靠近浮标区域慢速航行。

### 11.7 保存地图失败

先确认 `/map` 存在：

```bash
ros2 topic echo --once /map
```

再保存：

```bash
ros2 run nav2_map_server map_saver_cli -f ~/yildiz_ws/maps/roboboat_manual_map
```

如果 `/map` 没有数据，说明 SLAM 没正常建图。

## 12. 本模式使用到的数据

GPS：

```text
/roboboat/sensors/gps/navsat
latitude / longitude / altitude
```

IMU：

```text
/roboboat/sensors/imu/imu
orientation
angular_velocity
linear_acceleration
```

LiDAR：

```text
/roboboat/sensors/lidar/scan
ranges[]
angle_min / angle_max / angle_increment
range_min / range_max
```

定位输出：

```text
/odometry/filtered
odom -> base_link
```

SLAM 输出：

```text
/map
map -> odom
```

手动控制输出：

```text
/roboboat/thrusters/left/thrust
/roboboat/thrusters/right/thrust
```
