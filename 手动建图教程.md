# RoboBoat 手动建图速查

## 1. 数据链路

正常建图不使用 `/sim/odometry`，不使用相机，不启动 Nav2。

```text
GPS: /roboboat/sensors/gps/navsat
  sensor_msgs/NavSatFix: latitude / longitude / altitude / covariance
  -> gps_covariance_repub -> /gps/fixed_cov
  -> navsat_transform_node -> /odometry/gps
  -> ekf_node

IMU: /roboboat/sensors/imu/imu
  sensor_msgs/Imu: orientation / angular_velocity / covariance
  EKF 只融合 yaw 和 yaw_rate
  -> imu_covariance_repub -> /imu/fixed_cov
  -> ekf_node

LiDAR: /roboboat/sensors/lidar/scan
  sensor_msgs/LaserScan: ranges[] / angles / range_min / range_max
  当前 15 m，前方约 270 度，后方约 90 度盲区
  -> slam_toolbox -> /map
```

```text
GPS + IMU -> /odometry/filtered -> odom -> base_link
LiDAR + TF -> slam_toolbox -> /map -> map -> odom
```

## 2. 每个终端先执行

```bash
cd ~/yildiz_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
```

改过代码后先构建：

```bash
colcon build --merge-install
source install/setup.bash
```

## 3. 清理旧进程

```bash
pkill -f "workspace_slam|slam_toolbox|rviz2|workspace_nav|workspace_ros|workspace_gz|gz sim|manual_control|robot_state_publisher|joint_state_publisher|static_transform_publisher" || true
ros2 daemon stop
ros2 daemon start
pgrep -af "rviz2|slam_toolbox|workspace_slam|workspace_gz|workspace_ros|workspace_nav|gz sim|map_server|amcl|robot_state_publisher|joint_state_publisher|static_transform_publisher"
```

最后一行没有输出最好。

## 4. 启动顺序

终端 1：Gazebo

```bash
export DISPLAY=:0
ros2 launch workspace_gz simulation.launch.py
```

终端 2：定位融合

```bash
ros2 launch workspace_ros localization.launch.py
```

终端 3：SLAM + RViz

```bash
ros2 launch workspace_slam slam.launch.py
```

只后台建图：

```bash
ros2 launch workspace_slam slam.launch.py rviz:=false
```

终端 4：手动控制

```bash
ros2 run workspace_ros manual_control
```

```text
W 前进 / A 左转 / D 右转 / S 停止 / Q 退出
```

## 5. 必查项

传感器：

```bash
ros2 topic list | grep roboboat/sensors
ros2 topic hz /roboboat/sensors/lidar/scan
```

定位：

```bash
ros2 topic info -v /odometry/filtered
ros2 topic echo --once /odometry/filtered
ros2 run tf2_ros tf2_echo odom base_link
```

应满足：

```text
/odometry/filtered publisher: ekf_node
frame_id: odom
child_frame_id: base_link
```

地图：

```bash
ros2 topic info -v /map
ros2 run tf2_ros tf2_echo map base_link
ros2 topic echo --once /map | sed -n '1,25p'
ros2 node list | grep map_to_odom
```

应满足：

```text
/map 只有 slam_toolbox 一个 publisher
TF 通路：map -> odom -> base_link
通常不应有额外 map_to_odom_tf
```

## 6. 航行建议

```text
慢速航行，少长按 W
靠近浮标、岸边、桥墩等 LiDAR 特征
不要长时间只在空旷水面直线跑
掉头前减速，回程低速
尽量形成闭环
```

## 7. 保存地图

```bash
mkdir -p ~/yildiz_ws/src/YILDIZ-USV/maps
ros2 run nav2_map_server map_saver_cli -f ~/yildiz_ws/src/YILDIZ-USV/maps/roboboat_manual_map
ls -lh ~/yildiz_ws/src/YILDIZ-USV/maps/roboboat_manual_map.*
```

生成：

```text
roboboat_manual_map.yaml
roboboat_manual_map.pgm
```

## 8. 快速判断问题

```text
地图重影：
  ros2 topic info -v /map
  ros2 node list | grep map_to_odom
  确认 /map 只有 slam_toolbox，且未启动 Nav2 / AMCL / map_server

RViz 和 Gazebo 位置不一致：
  odom -> base_link 漂：查 GPS / IMU / EKF
  map -> base_link 漂：查 LiDAR 特征 / slam_toolbox

地图不增长：
  查 /roboboat/sensors/lidar/scan
  查 /map publisher
  查 TF: map -> base_link
```
